# TODO Template

## 使用要求

- 所有开发任务都应维护 TODO。
- TODO 需要可执行、可验证、可追踪。
- 任务按优先级分为 `P0`、`P1`、`P2`。
- 每项任务要明确新增、修改或删除的对象。

## 推荐模板

```md
# Task

## 背景

- 说明当前问题或目标。

## 目标

- 说明本次交付的完成标准。

## 范围

- 说明本次会改什么。
- 说明本次不会改什么。

## TODO

### P0
- [ ] 任务 1
- [ ] 任务 2

### P1
- [ ] 任务 3

### P2
- [ ] 任务 4

## 验证

- [ ] 编译检查
- [ ] 测试检查
- [ ] 结果检查

## 风险

- 列出已知风险和后续事项。
```

## 维护规则

- 需求变化时先更新 TODO，再改实现。
- 完成一项就同步状态，避免 TODO 与实际进度脱节。
- TODO 应和目录设计保持一致，避免任务描述跨越模块边界。
